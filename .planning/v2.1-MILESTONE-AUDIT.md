## Cross-Phase Integration

**Score:** 24/25 integration paths wired correctly

### Integration Gap (Critical)

**PROV-05: wrapModel/routerModel broken by async-only registration**

- **From:** `ProviderRegistry.createDefault()` in `src/wrapper/provider-registry.ts` (Phase 19-02)
- **To:** `createProviderInstance` (sync) in `src/config/index.ts:365` and `src/wrapper/router-model.ts:34`
- **Issue:** Phase 19-02 replaced all `registry.register()` (sync) calls with `registry.registerAsync()` in `createDefault()`. The sync `factories` map is now always empty. `wrapModel()` and `routerModel()` call sync `createProviderInstance` → `registry.get(provider)` → finds nothing → throws `ConfigError`.
- **Impact:** `router.wrapModel("google/gemini-2.0-flash")` throws at runtime. The primary `router.chat()` chain path is unaffected (uses `createProviderInstanceAsync`).
- **Fix:** Change `wrapModel` and `routerModel` to call `createProviderInstanceAsync` instead of `createProviderInstance`, matching the Phase 17-02 fix pattern in `retry-proxy.ts`.
