   ```typescript
   static async createDefault(): Promise<ProviderRegistry> {
     const registry = new ProviderRegistry();

     // Register all active provider modules (handles missing SDKs gracefully via createFactory)
     const { getAllProviders } = await import('../providers/registry.js');
     for (const mod of getAllProviders()) {
       registry.registerAsync(mod.id, mod.createFactory.bind(mod));
     }

     return registry;
   }
   ```

   This replaces the old implementation that only loaded `@ai-sdk/google` with a loop over all 6 provider modules from the registry. The `createFactory.bind(mod)` matches the `AsyncProviderFactory` type signature `(apiKey: string) => Promise<(modelId: string) => LanguageModelV3>`. Each provider module's `createFactory` already handles missing SDK packages gracefully (try/catch with ConfigError).

   Note: The old `createDefault()` used `registry.register()` (sync). The new one uses `registry.registerAsync()` because provider modules expose async `createFactory`. This is correct -- `createProviderInstanceAsync()` already checks async factories first.
