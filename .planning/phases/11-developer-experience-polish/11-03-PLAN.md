**docs/troubleshooting.md:**

Write a troubleshooting guide organized by symptom/error type. Per CONTEXT.md, cover both config mistakes and runtime issues.

Structure:

1. **Config Errors** (at createRouter time):
   - "At least one provider is required" -- empty providers object. Show fix.
   - "At least one key is required" -- provider with empty keys array. Show fix.
   - "Cannot use 'cheapest-paid' fallback with $0 budget" -- cross-field validation error. Show fix (increase budget or change behavior).
   - "Provider 'X' not recognized. Did you mean 'Y'?" -- typo in provider name. Show fix.
   - Unknown config fields with `.strict()` -- extra keys in config. Show fix.
   - "Model ID must be in provider/model format" -- missing slash in modelId. Show fix.

2. **Runtime Errors:**
   - `AllProvidersExhaustedError` -- all keys exhausted. Causes: hit rate limits, quota exhausted, all keys disabled. Solutions: add more keys, add more providers, enable fallback, check usage with `router.getUsage()`.
   - `AuthError` (401/403) -- bad API key. Solutions: check key is valid, check env var is set, check provider dashboard.
   - `RateLimitError` (429) -- hit rate limit. Note: router retries automatically with key rotation. If still failing: all keys hit limits simultaneously. Solutions: add keys, reduce request rate.
   - `QuotaExhaustedError` -- monthly/daily quota used. Solutions: wait for reset, add more keys, enable paid fallback.
   - `ProviderError` (500+) -- provider server error. Usually transient. Router retries automatically.
   - `NetworkError` -- connection failure. Check network, check provider status page.

3. **Storage Adapter Issues:**
   - "Cannot find module 'better-sqlite3'" -- missing peer dep. Fix: `npm install better-sqlite3`.
   - "Cannot find module 'ioredis'" -- missing peer dep. Fix: `npm install ioredis`.
   - Redis connection refused -- Redis not running or wrong host/port. Show connection config.
   - SQLite permission error -- check directory exists and is writable.

4. **Debug Tips:**
   - Enable debug mode: `createRouter({ ..., debug: true })` or `DEBUG=llm-router:* node app.js`
   - Check resolved config: `router.getConfig()`
   - Check current usage: `await router.getUsage()`
   - Check provider usage: `await router.getUsage('google')`
   - Reset usage counters: `await router.resetUsage()`
