Class `SqliteStorage implements StorageBackend`:

- Private constructor accepting `db: Database` instance
- Static async `create(options?: SqliteStorageOptions): Promise<SqliteStorage>` factory:
  1. Dynamic `import('better-sqlite3')` in try/catch -- throw `PennyLLMError` with code `MISSING_PEER_DEPENDENCY` and message "better-sqlite3 is required for SqliteStorage. Install it: npm install better-sqlite3" on failure
  2. Resolve db path: `options?.path ?? getDefaultDbPath()`
  3. `mkdirSync(dirname(dbPath), { recursive: true })` to ensure parent directory exists (prevents SQLITE_CANTOPEN on first run)
  4. Open database with `new BetterSqlite3(dbPath)`
  5. Enable WAL mode: `db.pragma('journal_mode = WAL')`
  6. Run `migrate(db)` for schema setup
  7. Prepare cached statements (upsert, get, getUsage, reset, delete)
  8. Return new SqliteStorage(db)

Prepared statements (cached as private fields):

- `upsertStmt`: INSERT ... ON CONFLICT DO UPDATE for usage_counters
- `getStmt`: SELECT by composite key
- `getUsageStmt`: SELECT prompt_tokens, completion_tokens, call_count by provider+keyIndex+windowType+periodKey
- `resetStmt`: DELETE by provider+keyIndex+windowType+periodKey
- `resetAllStmt`: DELETE from usage_counters (with optional provider/keyIndex WHERE clause -- use separate statements for provider-only, provider+keyIndex, and all)
- `getByProviderStmt`: SELECT \* WHERE provider = ?
- `cleanupStmt`: DELETE FROM usage_counters WHERE window_type = ? AND period_key < ?

Method implementations (all methods use `eslint-disable-next-line @typescript-eslint/require-await` since better-sqlite3 is synchronous):

`increment()`:

1. Call `this.ensureOpen()`
2. Call `this.cleanupExpired(window)` for lazy cleanup
3. Get periodKey via `getPeriodKey(window, Date.now())`
4. Run upsert statement with parameters
5. Read back updated values with getStmt
6. Return UsageRecord with composite key as `id` field: `${provider}:${keyIndex}:${window.type}:${periodKey}`
7. The id must be deterministic so contract test `record2.id === record1.id` passes for same-key increments

`getUsage()`:

1. Call `this.ensureOpen()`
2. Get periodKey via `getPeriodKey(window, Date.now())`
3. Query by composite key
4. Return `{ promptTokens, completionTokens, totalTokens, callCount }` (zeros if not found)

`get(key)`:

1. Call `this.ensureOpen()`
2. Parse key as provider name (following MemoryStorage convention where get() retrieves by provider)
3. Query all rows WHERE provider = key
4. Map rows to UsageRecord[] (reconstruct TimeWindow from window_type column, set durationMs based on window type)

`put(record)`:

1. Call `this.ensureOpen()`
2. Get periodKey from record.window and record.timestamp
3. Upsert the record (same as increment but with explicit values, not additive -- use a separate REPLACE statement or DELETE+INSERT)

`reset(provider, keyIndex, window)`:

1. Call `this.ensureOpen()`
2. Get periodKey via `getPeriodKey(window, Date.now())`
3. DELETE WHERE provider AND key_index AND window_type AND period_key

`resetAll(provider?, keyIndex?)`:

1. Call `this.ensureOpen()`
2. If no args: DELETE all from usage_counters
3. If provider only: DELETE WHERE provider = ?
4. If provider + keyIndex: DELETE WHERE provider = ? AND key_index = ?

`close()`:

1. Close the database via `db.close()`
2. Set `closed = true`
3. Clear statement references

`ensureOpen()`:

- Throw `PennyLLMError` with code `STORAGE_CLOSED` if `closed === true`

`cleanupExpired(window)`:

- Delete rows WHERE window_type = window.type AND period_key < currentPeriodKey
- For per-minute: compare numeric period keys
- For calendar windows (hourly/daily/monthly/rolling-30d): compare string period keys lexicographically (ISO date strings sort correctly)

Debug logging: Use `debug('pennyllm:sqlite')` namespace for key operations (create, increment, close).
