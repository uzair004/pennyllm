**1. Create src/selection/strategies/priority.ts:**

```typescript
export class PriorityStrategy implements SelectionStrategy {
  readonly name = 'priority';

  async selectKey(context: SelectionContext): Promise<SelectionResult> {
    // Filter to eligible, non-cooldown candidates
    const available = context.candidates.filter((c) => c.eligible && !c.cooldown);

    if (available.length === 0) {
      throw new Error('No eligible keys');
    }

    // Pre-flight headroom check if estimatedTokens provided
    if (context.estimatedTokens !== undefined) {
      const withHeadroom = available.filter((c) => {
        // Check if any token limit has enough remaining
        const tokenLimits = c.evaluation.limits.filter((l) => l.type === 'tokens');
        if (tokenLimits.length === 0) return true; // No token limits = always OK
        return tokenLimits.every((l) => l.remaining >= context.estimatedTokens!);
      });
      if (withHeadroom.length > 0) {
        // Use first with headroom (config order)
        const selected = withHeadroom[0];
        return {
          keyIndex: selected.keyIndex,
          reason: 'first eligible with headroom in config order',
        };
      }
      // No key has enough headroom — pick first eligible anyway (advisory, not blocking)
    }

    // First eligible key in config order (candidates are already in config order)
    const selected = available[0];
    return { keyIndex: selected.keyIndex, reason: 'first eligible in config order' };
  }
}
```

Note: auto-promotion is implicit — priority re-evaluates every request, so a recovered key (Key 0) naturally gets picked first again.

```typescript
export class RoundRobinStrategy implements SelectionStrategy {
  readonly name = 'round-robin';
  private indices = new Map<string, number>();

  async selectKey(context: SelectionContext): Promise<SelectionResult> {
    const available = context.candidates.filter((c) => c.eligible && !c.cooldown);

    if (available.length === 0) {
      throw new Error('No eligible keys');
    }

    const currentIndex = this.indices.get(context.provider) ?? 0;
    const selected = available[currentIndex % available.length];

    // Advance for next call
    this.indices.set(context.provider, (currentIndex + 1) % available.length);

    return {
      keyIndex: selected.keyIndex,
      reason: `round-robin position ${currentIndex % available.length}`,
    };
  }
}
```

In-memory state, resets on restart (as specified).

**3. Create src/selection/strategies/least-used.ts:**

```typescript
export class LeastUsedStrategy implements SelectionStrategy {
  readonly name = 'least-used';

  async selectKey(context: SelectionContext): Promise<SelectionResult> {
    const available = context.candidates.filter((c) => c.eligible && !c.cooldown);

    if (available.length === 0) {
      throw new Error('No eligible keys');
    }

    // Sort by worst-case remaining percentage (descending — most remaining first)
    // Tiebreaker: first in config order (lower keyIndex wins)
    const sorted = [...available].sort((a, b) => {
      const aRemaining = this.getWorstCaseRemainingPct(a);
      const bRemaining = this.getWorstCaseRemainingPct(b);
      if (bRemaining !== aRemaining) {
        return bRemaining - aRemaining; // Higher remaining first
      }
      return a.keyIndex - b.keyIndex; // Tiebreaker: config order
    });

    const selected = sorted[0];
    const pct = this.getWorstCaseRemainingPct(selected);

    return {
      keyIndex: selected.keyIndex,
      reason: `most remaining quota (${pct.toFixed(1)}% worst-case)`,
    };
  }

  private getWorstCaseRemainingPct(candidate: CandidateKey): number {
    const limits = candidate.evaluation.limits;
    if (limits.length === 0) return 100; // No limits = 100% remaining

    let worstPct = 100;
    for (const limit of limits) {
      if (limit.max > 0) {
        const pct = (limit.remaining / limit.max) * 100;
        worstPct = Math.min(worstPct, pct);
      }
    }
    return worstPct;
  }
}
```

**1. Create src/selection/KeySelector.ts:**

KeySelector orchestrates: strategy resolution, candidate evaluation, cooldown checking, strategy execution, and event emission.

Constructor:

```typescript
constructor(
  private config: RouterConfig,
  private policyEngine: PolicyEngine,
  private cooldownManager: CooldownManager,
  private emitter: EventEmitter,
  private customStrategy?: SelectionStrategy | ((context: SelectionContext) => SelectionResult | Promise<SelectionResult>),
)
```

- Create built-in strategy instances internally: `PriorityStrategy`, `RoundRobinStrategy`, `LeastUsedStrategy`. Store in a `Map<string, SelectionStrategy>`.
- Store in-memory selection metrics: `selectionCounts: Map<string, number>` (key: `${provider}:${keyIndex}`, value: count).

**Public method `selectKey(provider: string, model?: string, options?: { strategy?: string; estimatedTokens?: number; requestId?: string }): Promise<SelectionResult>`:**

1. Get provider config from `this.config.providers[provider]`. If not found, throw new Error(`Provider ${provider} not configured`).

2. Get keys array from provider config.

3. **Single-key short-circuit:** If keys.length === 1:
   - Check cooldown: `this.cooldownManager.getCooldown(provider, 0)`. If in cooldown, throw `RateLimitError`.
   - Check eligibility: `await this.policyEngine.evaluate(provider, 0, estimatedTokens as { prompt, completion } if number convert to { prompt: estimatedTokens, completion: 0 })`. If not eligible, throw `QuotaExhaustedError`.
   - Emit 'key:selected' event, increment metrics, return `{ keyIndex: 0, reason: 'only key available' }`.

4. **Build candidate list:**

   ```typescript
   const candidates: CandidateKey[] = [];
   for (let i = 0; i < keys.length; i++) {
     const keyConfig = keys[i];
     const label = typeof keyConfig === 'object' ? keyConfig.label : undefined;
     const cooldown = this.cooldownManager.getCooldown(provider, i);
     const evaluation = await this.policyEngine.evaluate(
       provider,
       i,
       estimatedTokens ? { prompt: estimatedTokens, completion: 0 } : undefined,
     );
     candidates.push({
       keyIndex: i,
       eligible: evaluation.eligible,
       cooldown,
       evaluation,
       ...(label !== undefined ? { label } : {}),
     });
   }
   ```

5. **Check if all exhausted/cooldown:** Count exhaustedCount (not eligible, no cooldown), cooldownCount (has cooldown). If no candidates are eligible AND not in cooldown:
   - Find earliest recovery: check cooldown.until dates and evaluation resetAt dates.
   - Emit 'provider:exhausted' event: `{ provider, totalKeys: keys.length, exhaustedCount, cooldownCount, earliestRecovery, timestamp: Date.now() }`.
   - If cooldownCount > 0 and exhaustedCount === 0: throw RateLimitError.
   - If exhaustedCount > 0: throw QuotaExhaustedError.
   - Mixed: throw QuotaExhaustedError (quota is the limiting factor).

6. **Resolve strategy:** `resolveStrategy(provider, options?.strategy)`.
   - Check: per-request override > per-provider config > global config default.
   - Per-request override can be a string (built-in) or the customStrategy provided at construction.
   - If custom strategy provided via options and it's a string matching a built-in, use built-in.
   - Return the SelectionStrategy instance.

7. **Execute strategy** with try-catch:

   ```typescript
   try {
     result = await strategy.selectKey({
       provider,
       model,
       candidates,
       estimatedTokens: options?.estimatedTokens,
     });
   } catch (strategyError) {
     // Custom strategy failed — fall back to provider default
     debug('Custom strategy error, falling back: %s', strategyError);
     const fallbackStrategy = this.getProviderStrategy(provider); // built-in only
     result = await fallbackStrategy.selectKey({
       provider,
       model,
       candidates,
       estimatedTokens: options?.estimatedTokens,
     });
   }
   ```

8. **Emit key:selected event:**

   ```typescript
   this.emitter.emit('key:selected', {
     provider,
     model,
     keyIndex: result.keyIndex,
     label: candidates.find((c) => c.keyIndex === result.keyIndex)?.label,
     strategy: strategy.name,
     reason: result.reason,
     timestamp: Date.now(),
     requestId: options?.requestId,
   });
   ```

9. **Increment selection metrics.** Return result.

**Public method `getSelectionStats()`:** Return a copy of selectionCounts as `Record<string, number>`.

**Public method `resetStats()`:** Clear selectionCounts.

**Private method `resolveStrategy(provider: string, override?: string): SelectionStrategy`:**

- If override is a string and matches built-in: return built-in.
- If customStrategy was provided and override === 'custom' or override is undefined and customStrategy exists and is not the default: wrap function into { name: 'custom', selectKey: fn } if it's a plain function.
- Per-provider: `this.config.providers[provider]?.strategy`. If set, return built-in.
- Global: `this.config.strategy`. Return built-in.

**Private method `getProviderStrategy(provider: string): SelectionStrategy`:** Same as resolveStrategy but without custom, only built-in based on config.

**Debug logging:** Use `debug('pennyllm:selection')`. Log each skipped key with reason (cooldown/exhausted). Log resolved strategy. Log selected key.
