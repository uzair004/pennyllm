    **1. src/constants/index.ts** — Const objects with `as const` (NOT TypeScript enums):
    ```
    Strategy = { ROUND_ROBIN: 'round-robin', LEAST_USED: 'least-used' } as const
    StrategyType = typeof Strategy[keyof typeof Strategy]

    Provider = { GOOGLE: 'google', GROQ: 'groq', OPENROUTER: 'openrouter', MISTRAL: 'mistral', HUGGINGFACE: 'huggingface', CEREBRAS: 'cerebras', DEEPSEEK: 'deepseek', QWEN: 'qwen', CLOUDFLARE: 'cloudflare', NVIDIA: 'nvidia', COHERE: 'cohere', GITHUB: 'github' } as const
    ProviderType = typeof Provider[keyof typeof Provider]

    RouterEvent = { KEY_SELECTED: 'key:selected', USAGE_RECORDED: 'usage:recorded', LIMIT_WARNING: 'limit:warning', LIMIT_EXCEEDED: 'limit:exceeded', FALLBACK_TRIGGERED: 'fallback:triggered', CONFIG_LOADED: 'config:loaded', ERROR: 'error' } as const
    RouterEventType = typeof RouterEvent[keyof typeof RouterEvent]

    LimitType = { TOKENS: 'tokens', CALLS: 'calls', RATE: 'rate', DAILY: 'daily', MONTHLY: 'monthly' } as const
    LimitTypeValue = typeof LimitType[keyof typeof LimitType]

    EnforcementBehavior = { HARD_BLOCK: 'hard-block', THROTTLE: 'throttle', SILENT_CHARGE: 'silent-charge' } as const
    EnforcementBehaviorType = typeof EnforcementBehavior[keyof typeof EnforcementBehavior]

    QualityTier = { FRONTIER: 'frontier', HIGH: 'high', MID: 'mid', SMALL: 'small' } as const
    QualityTierType = typeof QualityTier[keyof typeof QualityTier]
    ```
    All values must be JSON-serializable strings. Export all constants and their type aliases.

    **2. src/types/domain.ts** — Domain types (all fields JSON-serializable):
    ```
    ModelMetadata — id: string, provider: string, name: string, capabilities: { reasoning: boolean, toolCall: boolean, structuredOutput: boolean, vision: boolean }, qualityTier: QualityTierType, contextWindow: number, pricing: { promptPer1kTokens: number, completionPer1kTokens: number } | null (null = free)

    Policy — id: string, provider: string, version: string, limits: PolicyLimit[], enforcement: EnforcementBehaviorType, resetWindows: ResetWindow[], metadata: { researchedDate: string, confidence: 'high' | 'medium' | 'low', sourceUrl?: string }

    PolicyLimit — type: LimitTypeValue, value: number, window: TimeWindow

    TimeWindow — type: 'per-minute' | 'hourly' | 'daily' | 'monthly' | 'rolling-30d', durationMs: number

    UsageRecord — id: string, provider: string, keyIndex: number, promptTokens: number, completionTokens: number, totalTokens: number, timestamp: number, window: TimeWindow, estimated: boolean

    ResetWindow — type: TimeWindow['type'], resetAt: 'calendar' | 'rolling'
    ```
    Export all types. Use types from constants (StrategyType, ProviderType, etc.) where appropriate.

    **3. src/types/config.ts** — Configuration types (these are the TypeScript types; Zod schema in Plan 02):
    ```
    RouterConfig — version: '1.0', providers: Record<string, ProviderConfig>, strategy: StrategyType, storage: StorageConfig, budget: BudgetConfig

    ProviderConfig — keys: string[], strategy?: StrategyType, limits?: PolicyLimit[] (override), enabled?: boolean

    StorageConfig — type: 'sqlite' | 'redis' | 'memory', path?: string (for sqlite), url?: string (for redis)

    BudgetConfig — monthlyLimit: number (0 = no paid calls), alertThresholds: number[] (e.g., [0.8, 0.95])
    ```
    These types should match what the Zod schema will produce. Export all config types.

    **4. src/types/interfaces.ts** — Three core interfaces:
    ```
    StorageBackend — Interface for persistence layer:
      - get(key: string): Promise<UsageRecord[]>
      - put(record: UsageRecord): Promise<void>
      - increment(provider: string, keyIndex: number, tokens: { prompt: number, completion: number }, window: TimeWindow): Promise<UsageRecord>
      - getUsage(provider: string, keyIndex: number, window: TimeWindow): Promise<number>
      - reset(provider: string, keyIndex: number, window: TimeWindow): Promise<void>
      - close(): Promise<void>

    ModelCatalog — Interface for model metadata:
      - getModel(modelId: string): Promise<ModelMetadata | null>
      - listModels(provider?: string): Promise<ModelMetadata[]>
      - getCapabilities(modelId: string): Promise<ModelMetadata['capabilities'] | null>
      - refresh(): Promise<void>

    SelectionStrategy — Interface for key selection:
      - selectKey(provider: string, availableKeys: string[], usage: Map<number, UsageRecord[]>): Promise<{ keyIndex: number, reason: string }>
      - name: string
    ```
    Keep interfaces minimal with few required methods per user decision on plugin DX. Import domain types from domain.ts.

    **5. src/types/events.ts** — Type-safe event map:
    ```
    RouterEventPayload — Base: { timestamp: number, requestId?: string }

    KeySelectedEvent — extends base + { provider: string, keyIndex: number, reason: string, remainingQuota?: number }
    UsageRecordedEvent — extends base + { provider: string, keyIndex: number, promptTokens: number, completionTokens: number }
    LimitWarningEvent — extends base + { provider: string, keyIndex: number, limitType: LimitTypeValue, currentUsage: number, limit: number, threshold: number }
    LimitExceededEvent — extends base + { provider: string, keyIndex: number, limitType: LimitTypeValue }
    FallbackTriggeredEvent — extends base + { fromProvider: string, toProvider: string, reason: string }
    ConfigLoadedEvent — extends base + { providerCount: number, keyCount: number }
    ErrorEvent — extends base + { error: PennyLLMError }

    RouterEventMap — Map from RouterEvent constant values to their event types (e.g., 'key:selected' -> KeySelectedEvent)
    ```
    Also export a wildcard event type that is the union of all event payloads.

    **6. src/types/index.ts** — Re-export everything from config.ts, domain.ts, interfaces.ts, events.ts.

    **7. src/errors/base.ts** — Per RESEARCH.md Pattern 6:
    PennyLLMError extends Error with: code (string), suggestion (optional string), metadata (optional Record<string, unknown>), cause (optional Error). Constructor accepts message + options object. Has toJSON() returning { name, code, message, suggestion, metadata, stack }. Uses Error.captureStackTrace.

    **8. src/errors/config-error.ts** — ConfigError extends PennyLLMError with code 'CONFIG_ERROR', suggestion 'Check your configuration against the schema'. Constructor accepts message + optional { field, cause }.

    **9. src/errors/index.ts** — Re-export PennyLLMError, ConfigError.

    - `src/selection/index.ts` — Re-export SelectionStrategy interface from types
    - `src/policy/index.ts` — Re-export Policy, PolicyLimit, ResetWindow types from types

    IMPORTANT: All types must be JSON-serializable (no Date objects, no functions in data types, no Map/Set). Use number for timestamps (Unix ms), string for dates.
