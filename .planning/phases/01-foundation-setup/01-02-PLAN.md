    **2. src/config/schema.ts** — Create Zod schema per RESEARCH.md Pattern 2:
    - `providerConfigSchema` — z.object with keys: z.array(z.string()).min(1), strategy: z.enum([Strategy.ROUND_ROBIN, Strategy.LEAST_USED]).optional(), limits: z.array(policyLimitSchema).optional(), enabled: z.boolean().default(true)
    - `storageConfigSchema` — z.object with type: z.enum(['sqlite', 'redis', 'memory']).default('sqlite'), path: z.string().optional(), url: z.string().optional()
    - `budgetConfigSchema` — z.object with monthlyLimit: z.number().nonnegative().default(0), alertThresholds: z.array(z.number().min(0).max(1)).default([0.8, 0.95])
    - `configSchema` — z.object with version: z.literal('1.0').default('1.0'), providers: z.record(z.string(), providerConfigSchema).refine(obj => Object.keys(obj).length > 0, 'At least one provider required'), strategy: z.enum([Strategy.ROUND_ROBIN, Strategy.LEAST_USED]).default(Strategy.ROUND_ROBIN), storage: storageConfigSchema.default({}), budget: budgetConfigSchema.default({})
    - Export `configSchema`, `providerConfigSchema`, and inferred types.
    - Also export `type ConfigInput = z.input<typeof configSchema>` for the input type (fields with defaults are optional).
    - Use `.strict()` on the top-level schema to reject unknown keys.

    **IMPORTANT Zod pitfall from RESEARCH.md Pitfall 4:** Do NOT use `.optional()` before `.default()`. Use `.default()` directly — field is optional in input, guaranteed in output.

    **3. src/config/loader.ts** — Config file loading:
    - `interpolateEnvVars(str: string): string` — Replace `${VAR}` patterns with process.env values. Throw ConfigError if env var is undefined. Do NOT log actual env var values in error messages (security).
    - `loadConfigFile(filePath: string): RouterConfig` — Read file, detect JSON or YAML by extension (.json, .yml, .yaml). For JSON: JSON.parse. For YAML: try to require('js-yaml') dynamically, throw ConfigError if not installed explaining it's an optional peer dependency. Apply interpolateEnvVars on raw string before parsing. Parse through configSchema.parse() and return validated config.
    - Use `debug('llm-router:config')` for logging load operations. Redact any key values in debug output.

    **4. src/config/define-config.ts** — Per RESEARCH.md Pattern 3:
    - `defineConfig(config: ConfigInput): ConfigInput` — Identity function that returns its input. Provides IDE autocomplete. Zero runtime cost.
    - Import ConfigInput type from schema.ts.
