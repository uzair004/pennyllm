**Types to export:**

```typescript
export type ErrorType = 'rate_limit' | 'auth' | 'server' | 'network' | 'unknown';

export interface ClassifiedError {
  type: ErrorType;
  statusCode?: number;
  retryAfter?: string; // From Retry-After response header
  message: string;
  original: unknown;
  retryable: boolean; // Whether the retry proxy should attempt retry with a different key
}

export interface AttemptRecord {
  keyIndex: number;
  errorType: string;
  statusCode?: number;
  providerMessage: string;
  retryAfter?: string;
}
```

**classifyError(error: unknown): ClassifiedError** - Pure function, no side effects:

1. Check for `APICallError` using `APICallError.isInstance(error)` (NEVER use `instanceof` -- see RESEARCH.md Pitfall 4). Import from `@ai-sdk/provider`.
   - 429 -> `{ type: 'rate_limit', statusCode: 429, retryAfter: error.responseHeaders?.['retry-after'], message: error.message, original: error, retryable: true }`
   - 401 or 403 -> `{ type: 'auth', statusCode, message: error.message, original: error, retryable: true }`
   - statusCode >= 500 -> `{ type: 'server', statusCode, message: error.message, original: error, retryable: false }`
   - Other APICallError statusCodes -> `{ type: 'unknown', statusCode, message: error.message, original: error, retryable: false }`

2. Check for network errors (Node.js system errors):
   - Check `(error as NodeJS.ErrnoException).code` for: ECONNREFUSED, ETIMEDOUT, ENOTFOUND, ECONNRESET, EAI_AGAIN
   - ALSO check `(error as any).cause?.code` for the same codes (Pitfall 7: Node.js fetch() throws TypeError with cause containing system error)
   - If match: `{ type: 'network', message, original: error, retryable: true }`

3. Fallback: `{ type: 'unknown', message: error instanceof Error ? error.message : String(error), original: error, retryable: false }`

Use conditional property inclusion for optional fields (retryAfter, statusCode) to satisfy exactOptionalPropertyTypes.

**shouldRetry(classified: ClassifiedError, triedKeys: Set<number>): boolean** - Pure function:

- `server` -> false (per user decision: server errors mean provider is down)
- `unknown` -> false
- `network` -> `triedKeys.size <= 1` (per user decision: retry once only for network errors)
- `rate_limit` -> true (try all available keys)
- `auth` -> true (try all available keys)
- Otherwise: `classified.retryable`

**buildFinalError(provider, modelId, classified, attempts, originalError):**
Returns the appropriate LLMRouterError subclass based on classified.type:

- rate_limit -> new ProviderError(provider, modelId, 'rate_limit', attempts, { cause })
- auth -> new ProviderError(provider, modelId, 'auth', attempts, { cause })
- server -> new ProviderError(provider, modelId, 'server', attempts, { cause })
- network -> new ProviderError(provider, modelId, 'network', attempts, { cause })
- unknown -> new ProviderError(provider, modelId, 'unknown', attempts, { cause })

Import error classes from `../errors/index.js`.

IMPORTANT: The function must throw LLMRouterError subclasses (NOT APICallError). This prevents double-retry with the AI SDK's own retry mechanism (Pitfall 1 from RESEARCH.md).

**makeAttemptRecord(keyIndex, classified): AttemptRecord** - Helper:
Build AttemptRecord from ClassifiedError, using conditional property inclusion for optional statusCode and retryAfter.

Update `src/wrapper/index.ts` to export: `classifyError`, `shouldRetry`, `buildFinalError`, `makeAttemptRecord`, and the types `ClassifiedError`, `AttemptRecord`, `ErrorType`.
