```typescript
  initialKeyIndex: number;
  initialKey: string; // API key string for initial selection

  requestId: string;
  keyIndexRef: { current: number }; // Mutable ref shared with middleware
  keySelector: KeySelector; // For re-selection on retry
}
```

**createRetryProxy(options: RetryProxyOptions): LanguageModelV3:**

Return a plain object implementing LanguageModelV3:

```typescript
const proxy: LanguageModelV3 = {
  specificationVersion: 'v3' as const,
  provider: 'llm-router',
  modelId: options.modelId,
  supportedUrls: options.initialModel.supportedUrls,

  async doGenerate(params) { ... },
  async doStream(params) { ... },
};
```

**doGenerate implementation:**

1. Track `startTime = Date.now()`, `attempts: AttemptRecord[] = []`, `triedKeys = new Set<number>()`, `currentModel`, `currentKeyIndex`
2. Loop:
   a. Add currentKeyIndex to triedKeys
   b. Try `currentModel.doGenerate(params)` -- params pass through UNCHANGED (preserves tool calling, structured output, all AI SDK features per INTG-02)
   c. On success:
   - Call `cooldownManager.onSuccess(provider, currentKeyIndex)` to reset consecutive failures
   - Update `keyIndexRef.current = currentKeyIndex` so middleware tracks usage against correct key
   - Emit `request:complete` event (fire-and-forget, try/catch around emit): `{ provider, modelId, keyIndex: currentKeyIndex, requestId, promptTokens, completionTokens, latencyMs: Date.now() - startTime, retries: attempts.length, timestamp: Date.now() }`. Extract tokens from result.usage using `Number(result.usage?.inputTokens?.total) || 0` pattern.
   - Return result
     d. On error:
   - `const classified = classifyError(error)`
   - `const attempt = makeAttemptRecord(currentKeyIndex, classified)`
   - Push to attempts array
   - Emit per-attempt error event (fire-and-forget): emit `error:rate_limit`, `error:auth`, `error:server`, or `error:network` based on classified.type. Payload: `{ provider, keyIndex: currentKeyIndex, modelId, statusCode: classified.statusCode, message: classified.message, requestId, timestamp: Date.now() }`. For unknown type, emit `error` generic event.
   - Handle key state:
     - If `classified.type === 'rate_limit'`: `cooldownManager.setCooldown(provider, currentKeyIndex, classified.retryAfter)`
     - If `classified.type === 'auth'`: `disabledKeys.add(\`${provider}:${currentKeyIndex}\`)`, emit `key:disabled`event:`{ provider, keyIndex: currentKeyIndex, reason: 'auth_failed', statusCode: classified.statusCode, timestamp: Date.now() }`
   - Check `shouldRetry(classified, triedKeys)`:
     - If false: `throw buildFinalError(provider, modelId, classified, attempts, error)`
   - Get next key. Use the following approach:
     - Try to re-select via `keySelector.selectKey(provider, modelName, { requestId })`. This returns a SelectionResult with keyIndex and reason.
     - Check if the returned keyIndex is already in `triedKeys` OR in `disabledKeys`. If so, no keys left -- throw buildFinalError.
     - Resolve the actual API key string from `config.providers[provider].keys[result.keyIndex]` (same pattern as router.model() in config/index.ts).
     - Create new model: `createProviderInstance(registry, provider, modelName, apiKey)`
   - Emit `key:retried` event (fire-and-forget): `{ provider, modelId, failedKeyIndex: currentKeyIndex, newKeyIndex: next.keyIndex, reason: classified.type, attempt: attempts.length, maxAttempts: totalAvailableKeys, requestId, timestamp: Date.now() }`. For maxAttempts, use `config.providers[provider].keys.length`.
   - Update currentModel and currentKeyIndex
   - Continue loop

**doStream implementation:**
Same retry logic as doGenerate for the SETUP phase (the doStream() call itself). Once doStream() returns a LanguageModelV3StreamResult, return it directly -- NO mid-stream retry (per user decision). The stream consumer receives errors naturally if the stream fails after setup.

Key differences from doGenerate:

- `const result = await currentModel.doStream(params)` -- retry if this throws
- Once it succeeds, return result immediately (no usage extraction here -- middleware handles that via TransformStream)
- On success emit `request:complete` with `promptTokens: 0, completionTokens: 0` (actual usage tracked by middleware on stream finish), retries count, latencyMs for setup only

**Error event emission pattern (fire-and-forget):**

```typescript
try {
  emitter.emit(eventName, payload);
} catch {
  // Fire-and-forget: never let event emission break the retry loop (Pitfall 6)
}
```

**Debug logging:**
Use `debug('llm-router:retry')` namespace. Log on each retry attempt, on final failure, on key disable.

**CRITICAL constraints:**

- NEVER throw APICallError from the proxy. Always throw LLMRouterError subclasses to prevent AI SDK double-retry (Pitfall 1).
- Proxy identity (`provider: 'llm-router'`, `modelId: options.modelId`) stays constant even after key switches (RESEARCH.md Pattern 5).
- All event emissions wrapped in try/catch (Pitfall 6).
- All params pass through unchanged to underlying model -- this is what preserves tool calling, structured output, etc. (INTG-02).
