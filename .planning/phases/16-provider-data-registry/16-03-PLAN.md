1. **CONTRIBUTING.md** — Step-by-step contributor guide with these sections:

**Header:** `# Contributing to Awesome Free LLM APIs`

**Adding a New Provider** section:

1.  Copy `providers/_template.json` to `providers/{slug}.json` (slug = lowercase, hyphens, matches the `id` field)
2.  Fill in all fields using the provider's official documentation
3.  Run `node scripts/validate.js` to check your data
4.  Run `node scripts/generate.js` to regenerate README.md and registry.json
5.  Commit all changed files (your JSON + generated README.md + registry.json)
6.  Open a pull request

**Updating Existing Provider Data** section:

1.  Edit the relevant `providers/{slug}.json` file
2.  Update `lastVerified` to today's date
3.  Run `node scripts/validate.js` and `node scripts/generate.js`
4.  Commit and open a PR describing what changed and why

**Data Sources** section: Explain where to find provider information:

- Provider's official API documentation
- Rate limit response headers (make a test call)
- Community reports and forums
- Set `verifiedBy` to `"manual"` unless you have automated verification

**Schema Reference** section: Brief description of each field in the provider JSON. Point to `schema.json` for the full definition.

**Guidelines** section:

- Text LLMs only (no image, audio, or embedding providers)
- Free tier data only (no paid-only providers)
- Verify data accuracy before submitting
- One provider per JSON file
- Filename must match the `id` field

**Freshness** section: Explain the staleness system:

- Green (<=30 days since lastVerified): recently verified
- Yellow (>30 days): may need re-verification
- Red (>90 days): likely outdated, needs update
- PRs that only update `lastVerified` after re-verification are welcome
