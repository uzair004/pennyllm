1. **schema.json** — JSON Schema Draft 2020-12 (`"$schema": "https://json-schema.org/draft/2020-12/schema"`). Define the provider file structure with these sections:

   Required top-level string fields: `id`, `name`, `status`, `signupUrl`, `docsUrl`, `lastVerified`.

   Enums:
   - `status`: `["active", "degraded", "discontinued"]`
   - `freeTier.type`: `["perpetual", "trial", "rate-limited"]`
   - `sdk.type`: `["official", "openai-compat", "community"]`
   - `rateLimitHeaders.format`: `["seconds", "epoch", "iso"]`
   - `models[].capabilities` items: `["reasoning", "vision", "tools", "structuredOutput"]`
   - `models[].tier`: `["frontier", "high", "mid", "small"]`
   - `verifiedBy`: `["manual", "automated"]`

   Required objects: `auth` (with required `envVar` string, optional `keyPrefix` string, optional `header` string), `freeTier` (with required `type` enum and required `limits` object, optional `notes` string, optional `credits` string, optional `expiresAfterDays` number), `sdk` (with required `package` string and required `type` enum, optional `baseUrl` string).

   `freeTier.limits` is an object with all properties optional and numeric: `rpm`, `rpd`, `tpd`, `tpm`, `rps`, `tokenMonthly`.

   Optional object: `rateLimitHeaders` (with optional string fields `remaining`, `reset`, `limit`, and optional `format` enum).

   Required array: `models` — each item has required `id` (string), `free` (boolean), `capabilities` (array of enum strings), `tier` (enum string), `contextWindow` (number), `maxOutputTokens` (number). Optional: `limits` object (same shape as `freeTier.limits`).

   Optional array: `notes` — array of strings.

   Required: `lastVerified` (string, pattern `^\d{4}-\d{2}-\d{2}$`), `verifiedBy` (enum).

   Keep the schema as documentation of the contract. Do NOT implement complex conditional validation (trial fields etc.) in JSON Schema — that goes in validate.js.

Both scripts use `'use strict'` and CommonJS (`require`). Zero npm dependencies — only `node:fs` and `node:path`.

1. **scripts/validate.js** — Validates all `providers/*.json` files (excluding files starting with `_`).

   Validation checks to implement:
   - Parse JSON (report invalid JSON with filename)
   - Required string fields: `id`, `name`, `status`, `signupUrl`, `docsUrl`, `lastVerified` — must be non-empty strings
   - `id` must match filename (e.g., `cerebras.json` must have `"id": "cerebras"`)
   - `status` must be one of `["active", "degraded", "discontinued"]`
   - `signupUrl` and `docsUrl` must start with `https://`
   - `lastVerified` must match pattern `YYYY-MM-DD` (regex: `/^\d{4}-\d{2}-\d{2}$/`)
   - `verifiedBy` must be one of `["manual", "automated"]`
   - `auth` must be an object with `envVar` as non-empty string
   - `freeTier` must be an object with `type` (enum `["perpetual", "trial", "rate-limited"]`) and `limits` (object)
   - `freeTier.limits` values must be numbers when present; allowed keys: `rpm`, `rpd`, `tpd`, `tpm`, `rps`, `tokenMonthly`
   - Conditional: if `freeTier.type === "trial"`, `freeTier.credits` (string) and `freeTier.expiresAfterDays` (number) should be present (warn, not error)
   - `sdk` must be an object with `package` (non-empty string) and `type` (enum `["official", "openai-compat", "community"]`)
   - Conditional: if `sdk.type === "openai-compat"`, `sdk.baseUrl` must be present and start with `https://`
   - `models` must be a non-empty array; each model must have: `id` (non-empty string), `free` (boolean), `capabilities` (array of strings from `["reasoning", "vision", "tools", "structuredOutput"]`), `tier` (enum `["frontier", "high", "mid", "small"]`), `contextWindow` (positive number), `maxOutputTokens` (positive number)
   - `notes` if present must be an array of strings
   - `rateLimitHeaders` if present: `format` must be one of `["seconds", "epoch", "iso"]`

   Output format: Print each file as `  filename.json: OK` or `  filename.json: FAILED` followed by indented error lines `    - field: error message`. Print summary at end: `N files validated, M errors`. Exit 0 on success, 1 on any error.

   Also accept a single filename argument to validate just one file: `node scripts/validate.js providers/cerebras.json`.

2. **scripts/generate.js** — Reads all provider JSON files, produces README.md and registry.json.

   Accept `--check` flag: when present, compare generated output to existing files and exit 1 if different (for CI-like verification without overwriting).

   **README.md generation:**
   - Title: `# Awesome Free LLM APIs`
   - Subtitle: `> A community-maintained registry of free tier LLM API providers.`
   - Provider count line: `> {N} providers | Machine-readable JSON | Auto-generated`
   - Blank line then: `**Freshness:** {green_circle} Verified <=30 days | {yellow_circle} Verified >30 days | {red_circle} Verified >90 days`
   - Comparison table sorted by provider name with columns: Provider (linked to signupUrl), Free Tier Type, RPM, Daily Limit, SDK Package (backticked), Freshness (emoji badge)
   - Daily Limit column: show `rpd` if present, else `tpd` formatted as `{N} TPD`, else `-`
   - Per-provider sections (`## {name}`) containing: status badge, auth env var, free tier details (type, limits as bullet list, notes), SDK setup (package, type, baseUrl if applicable), models table (ID, Free, Tier, Capabilities, Context Window, Max Output), rate limit headers if present, notes as bullet list, lastVerified date
   - Usage examples section (`## Usage`) with 3 code snippets: (a) fetch registry.json via `fetch()`, (b) filter free models from a provider, (c) find providers with reasoning models
   - Contributing section linking to CONTRIBUTING.md
   - Footer: `---` then `Generated by [generate.js](scripts/generate.js) on {ISO date}. Do not edit manually.`

   **Freshness badge function:** Calculate days since lastVerified. `<=30` = green circle emoji (U+1F7E2), `<=90` = yellow circle (U+1F7E1), `>90` = red circle (U+1F534).

   **registry.json generation:**

   ```json
   {
     "$schema": "./schema.json",
     "version": "YYYY-MM",
     "generatedAt": "ISO timestamp",
     "providerCount": N,
     "providers": { "provider-id": { ...full provider data... } }
   }
   ```

   Providers keyed by `id` for O(1) lookup. Write with 2-space indent and trailing newline.
