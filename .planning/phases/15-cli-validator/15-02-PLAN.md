Create `src/cli/validate.ts` with the following exports:

**Main export: `runValidate(options: ValidateOptions): Promise<ValidationResult>`**

Implementation flow:

**Step 1: Load and validate config**

```typescript
import { resolveConfig } from './config-discovery.js';
import { configSchema } from '../config/schema.js';
import { buildChain } from '../chain/chain-builder.js';
import { getProviderModule } from '../providers/registry.js';
import { classifyError } from '../wrapper/error-classifier.js';
import type {
  ValidateOptions,
  ValidationResult,
  ModelTestResult,
  KeyTestResult,
  TestStatus,
} from './types.js';
import type { ChainEntry } from '../chain/types.js';
import type { RouterConfig } from '../types/config.js';
```

Call `resolveConfig({ config: options.config })` to get config and configPath.
Parse config through `configSchema` (resolveConfig already does this).

**Step 2: Build chain and group entries by provider**
Call `buildChain(config)` to get chain entries.
Group chain entries by provider: `Map<string, ChainEntry[]>`.

**Step 3: Filter providers if --provider flag used**
If `options.provider` is defined and non-empty, filter to only those provider IDs. If a requested provider is not in the chain, add a ModelTestResult with status 'fail' and message 'Provider not configured'.

**Step 4: For each provider, determine which model to test**
Per CONTEXT.md: "One model per provider — Test first model per provider to validate key + connectivity."
Pick the first chain entry for each provider as the tested model. All other chain entries for same provider get `tested: false` with overallStatus derived from key results.

**Step 5: --dry-run mode**
If `options.dryRun`, skip real API calls. For each provider:

- Check if provider module exists: `getProviderModule(providerId)`
- If missing: status 'fail', message `Missing ${mod.sdkPackage} — run: npm install ${mod.sdkPackage}`
- Try `mod.createFactory('dry-run-test-key')` to verify SDK loads
- If factory creation fails with MODULE_NOT_FOUND: status 'fail' with install command
- If factory creation succeeds: status 'pass', message 'Config valid (dry-run)'
- Return early with results

**Step 6: Real API test calls — parallel providers, sequential keys**
For each provider (in parallel via `Promise.allSettled`):

```typescript
async function testProvider(
  providerId: string,
  config: RouterConfig,
  testedEntry: ChainEntry,
  timeoutMs: number,
): Promise<KeyTestResult[]>;
```

Inside testProvider:

- Get provider module: `getProviderModule(providerId)`
- If no module: return single KeyTestResult with fail
- Get keys from `config.providers[providerId].keys`
- For each key (sequentially — for loop, not Promise.all):

```typescript
async function testKey(
  providerId: string,
  apiModelId: string,
  apiKey: string,
  keyIndex: number,
  timeoutMs: number,
): Promise<KeyTestResult>;
```

Inside testKey:

- Get provider module, call `mod.createFactory(apiKey)` to get factory
- Call `factory(apiModelId)` to get model instance
- Test generateText:
  ```typescript
  const controller = new AbortController();
  const timer = setTimeout(() => controller.abort(), timeoutMs);
  try {
    const { generateText } = await import('ai');
    const start = Date.now();
    const { text } = await generateText({
      model,
      prompt: 'Respond with the word ok.',
      maxTokens: 5,
      abortSignal: controller.signal,
    });
    clearTimeout(timer);
    const latencyMs = Date.now() - start;
    // pass if non-empty response
    generateResult = { status: 'pass' as TestStatus, latencyMs };
  } catch (err) {
    clearTimeout(timer);
    generateResult = classifyTestError(err, providerId, timeoutMs);
  }
  ```
- Test streamText (same pattern but consume stream):
  ```typescript
  const controller2 = new AbortController();
  const timer2 = setTimeout(() => controller2.abort(), timeoutMs);
  try {
    const { streamText } = await import('ai');
    const start = Date.now();
    const result = streamText({
      model,
      prompt: 'Respond with the word ok.',
      maxTokens: 5,
      abortSignal: controller2.signal,
    });
    let text = '';
    for await (const chunk of result.textStream) {
      text += chunk;
    }
    clearTimeout(timer2);
    const latencyMs = Date.now() - start;
    streamResult = { status: 'pass' as TestStatus, latencyMs };
  } catch (err) {
    clearTimeout(timer2);
    streamResult = classifyTestError(err, providerId, timeoutMs);
  }
  ```
- Return `{ keyIndex, generateText: generateResult, streamText: streamResult }`

**Step 7: Error classification helper**

```typescript
function classifyTestError(
  err: unknown,
  providerId: string,
  timeoutMs: number,
): { status: TestStatus; latencyMs?: number; message?: string };
```

- If AbortError (err.name === 'AbortError'): return `{ status: 'warning', message: \`Timeout (${timeoutMs / 1000}s) -- provider may be unreachable\` }`
- Use `classifyError(err)` from error-classifier:
  - `rate_limit` (429): return `{ status: 'warning', message: 'Rate limited (429) -- key valid, try again later' }`
  - `auth` (401/403): Get provider module for guidance URL. return `{ status: 'fail', message: \`${classified.statusCode} ${classified.message} -- Check: ${mod?.updateUrl ?? 'provider dashboard'}\` }`
  - `network`: return `{ status: 'warning', message: 'Network error -- provider may be unreachable' }`
  - Default: return `{ status: 'fail', message: classified.message }`

**Step 8: Assemble results**

For each provider, build `ModelTestResult[]`:

- Tested model: set `tested: true`, keys from test results, overallStatus = worst status across all keys (fail > warning > pass)
- Untested models (same provider): set `tested: false`, overallStatus based on tested model's key results ('pass' if keys work, 'skipped' otherwise), message '-- key ok' if keys passed

Compute summary: count unique providers, total keys, passed/failed/warning counts.

**Step 9: Handle createFactory errors gracefully**

Wrap `mod.createFactory(apiKey)` in try/catch:

- If error message contains 'MODULE_NOT_FOUND' or error code is 'MODULE_NOT_FOUND': return fail with `Missing ${mod.sdkPackage} -- run: npm install ${mod.sdkPackage}`
- Otherwise: return fail with error message

**Step 10: Compute exit code**

Add helper: `getExitCode(result: ValidationResult): number`

- If `result.summary.failed > 0`: return 1
- If `result.summary.warnings > 0`: return 2
- Return 0

Export `runValidate` and `getExitCode`.
