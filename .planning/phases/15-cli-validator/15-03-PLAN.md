function statusColor(status: TestStatus): (text: string) => string {
  switch (status) {

**`formatTable(result: ValidationResult, verbose: boolean): string`**

Build a manually-formatted table with these columns:

- Provider (left-aligned, padEnd)
- Keys (e.g., "2/2 keys ok" or per-key rows in verbose)
- Model (apiModelId)
- Tier (free/trial/paid)
- Status (colored PASS/FAIL/WARN/SKIP)
- Latency (e.g., "1.2s" from first key's generateText latency)
- Message (error message if applicable)

Implementation:

1. Define column widths: Provider=12, Keys=14, Model=30, Tier=6, Status=6, Latency=8, Message=rest
2. Print header row with `pc.bold()`: `Provider      Keys           Model                          Tier   Status  Latency  Message`
3. Print separator: `pc.dim('─'.repeat(totalWidth))`
4. For each ModelTestResult, sorted by provider name:
   - If NOT verbose (default): single row per model
     - Keys column: `${passedKeys}/${totalKeys} keys ok` (or `keys fail` if any failed)
     - Latency: average of generateText latencies, formatted as `${(ms/1000).toFixed(1)}s`
     - Message: first error/warning message from keys, or empty
   - If verbose: one row per key
     - First row shows Provider, Model, Tier
     - Subsequent key rows show `  Key ${i+1}` in Provider column, individual latency and message
     - Show both generateText and streamText results per key

For untested models (same provider, different model): show with Status='--' and Message='key ok' (if provider keys passed) or 'key fail' (if not).

5. After all rows, blank line + summary line

**`formatSummary(result: ValidationResult): string`**

Color the counts: green for passed, red for failed (if > 0), yellow for warnings (if > 0).

**Spinner management:**

```typescript
import { createSpinner } from 'nanospinner';
import type { Spinner } from 'nanospinner';

export interface SpinnerManager {
  start(providerId: string, providerName: string): void;
  success(providerId: string, message: string): void;
  error(providerId: string, message: string): void;
  warn(providerId: string, message: string): void;
  clear(): void;
}

export function createSpinnerManager(enabled: boolean): SpinnerManager;
```

If `enabled` is false (--json mode or non-TTY), return no-op implementations.
If enabled:

- `start()`: create and start spinner with text `Testing ${providerName}...`
- `success()`: call spinner.success({ text: message })
- `error()`: call spinner.error({ text: message })
- `warn()`: update spinner text with yellow prefix (nanospinner doesn't have warn — use `.update({ text: pc.yellow('!') + ' ' + message })` then `.stop()`)
- `clear()`: stop all active spinners
- Track active spinners in a Map<string, Spinner>

2. In the validate subcommand handler, replace the current raw JSON output with:

```typescript
// Create spinner manager (disabled for --json or non-TTY)
const spinners = createSpinnerManager(!opts.json && process.stdout.isTTY === true);

try {
  const result = await runValidate(opts, {
    onProviderStart: (id, name) => spinners.start(id, name),
    onProviderDone: (id, status, msg) => {
      if (status === 'pass') spinners.success(id, msg);
      else if (status === 'warning') spinners.warn(id, msg);
      else spinners.error(id, msg);
    },
  });

  // Clear any remaining spinners
  spinners.clear();

  // Output results
  if (opts.json) {
    console.log(formatJson(result));
  } else {
    console.log(formatTable(result, opts.verbose));
    console.log();
    console.log(formatSummary(result));
  }

  process.exitCode = getExitCode(result);
} catch (err) {
  spinners.clear();
  if (err instanceof Error) {
    console.error(`Error: ${err.message}`);
  } else {
    console.error(`Error: ${String(err)}`);
  }
  process.exitCode = 1;
}
```
