**Create `src/fallback/FallbackResolver.ts`**:

Class that resolves fallback candidates from the model catalog based on capabilities, quality tier, and cost.

**Constructor:**

```typescript
constructor(
  catalog: ModelCatalog,
  config: RouterConfig,
  emitter: EventEmitter,
)
```

Store all three. Create a `debug` logger namespaced `llm-router:fallback`.

**Method: `inferCapabilities(params: unknown, reasoning: boolean): Partial<ModelMetadata['capabilities']>`**

Static/standalone function that infers required capabilities from request parameters:

- If `params` has a `tools` property that is a non-empty array, set `toolCall: true`
- Check if the prompt/messages contain image content (look for `type: 'image'` in message parts). If found, set `vision: true`
- The `reasoning` parameter comes from config or per-request option. If true, set `reasoning: true`
- Return the partial capabilities object (only keys that are `true`)

NOTE: The actual shape of `params` varies by AI SDK version. Use defensive access: `(params as Record<string, unknown>).tools` etc. The function should never throw on unexpected shapes.

**Method: `resolve(originalModelId: string, requiredCaps: Partial<ModelMetadata['capabilities']>, options: ResolveOptions): Promise<FallbackCandidate[]>`**

Where `ResolveOptions` is:

```typescript
interface ResolveOptions {
  configuredProviders: string[]; // provider names with configured keys
  excludeProviders: string[]; // providers already tried
  budgetRemaining: number | null; // null = no budget tracking
  estimatedTokens?: number; // for context window check
  originalQualityTier?: QualityTierType;
}
```

Implementation steps:

1. Check if fallback is `enabled` in config. If not, return empty array.

2. Check `strictModel` in config. If true, return empty array (no fallback allowed).

3. Check `modelMappings` in config: if `originalModelId` has a mapping, check if the mapped model's provider is in `configuredProviders` and not in `excludeProviders`. If so, return it as the sole candidate.

4. **Tier 1 query:** Call `catalog.listModels({ capabilities: requiredCaps, qualityTier: options.originalQualityTier })` to get capability + same quality tier matches.

5. **Filter to configured providers only:** `candidates.filter(m => configuredProviders.includes(m.provider) && !excludeProviders.includes(m.provider) && m.id !== originalModelId)`.

6. **Tier 2 fallback:** If Tier 1 yields no results after filtering, call `catalog.listModels({ capabilities: requiredCaps })` (any quality tier). Apply same configured provider filter.

7. **Context window check:** If `estimatedTokens` is provided, filter out models where `contextWindow < estimatedTokens`.

8. **Rank candidates:**
   - Determine if model is free: `pricing !== null && pricing.promptPer1MTokens === 0 && pricing.completionPer1MTokens === 0`. Models with `pricing: null` are treated as "unknown" (sorted between free and paid).
   - Sort: free first, then unknown pricing, then cheapest paid.
   - Within free models: prefer models from providers with most remaining quota. Since FallbackResolver doesn't have direct access to usage data, sort free models by provider name for deterministic ordering (the FallbackProxy will handle actual quota-based selection among candidates from same provider).
   - Within paid models: sort by `pricing.promptPer1MTokens` ascending (cheapest first).

9. If `budgetRemaining !== null && budgetRemaining <= 0`, filter out all paid models (only return free candidates).

10. Convert `ModelMetadata[]` to `FallbackCandidate[]` format:

    ```typescript
    {
      provider: model.provider,
      modelId: model.id,
      modelName: model.id.substring(model.id.indexOf('/') + 1),
      qualityTier: model.qualityTier,
      capabilities: model.capabilities,
      pricing: model.pricing,
      contextWindow: model.contextWindow,
      isFree: isFreeModel(model),
    }
    ```

11. Limit to reasonable number of candidates (e.g., 20) to avoid excessive iteration.

**Helper: `isFreeModel(model: ModelMetadata): boolean`**

Returns true if `pricing !== null && pricing.promptPer1MTokens === 0 && pricing.completionPer1MTokens === 0`.

**Create `src/budget/BudgetTracker.ts`**:

Class that tracks monthly spend using StorageBackend and emits budget events.

**Constructor:**

```typescript
constructor(
  storage: StorageBackend,
  budgetConfig: BudgetConfig,
  emitter: EventEmitter,
)
```

- Store all three.
- Create a `debug` logger namespaced `llm-router:budget`.
- Initialize `alertsFired: Set<number>` to track which thresholds have already emitted alerts (prevents duplicate events per the deduplication pattern from Phase 3).
- Initialize `paidCallCount: number = 0` for avgCostPerRequest calculation (local counter, not persisted).

**Budget storage key pattern** (per research):

- Key: `budget` as provider name, `0` as keyIndex
- Window: `{ type: 'monthly', durationMs: 30 * 24 * 60 * 60 * 1000 }`
- Store cost as micro-dollars in the promptTokens field: `Math.round(cost * 1_000_000)`
- completionTokens field unused (0)
- callCount tracks number of paid calls

This repurposes the existing StorageBackend interface without adding new methods.

**Method: `async recordCost(provider: string, modelId: string, usage: { promptTokens: number; completionTokens: number }, pricing: ModelMetadata['pricing']): Promise<void>`**

Fire-and-forget cost recording:

1. If `pricing` is null: emit a warning debug log `'Model %s has no pricing data, cost not tracked'` and return. Do NOT emit a warning event here -- the FallbackProxy will handle the event. Just return silently.
2. If pricing is all zeros (free model): return immediately. Free calls are NOT tracked in budget.
3. Calculate cost: `(usage.promptTokens * pricing.promptPer1MTokens + usage.completionTokens * pricing.completionPer1MTokens) / 1_000_000`
4. If cost is 0 or negative, return (guard against bad data).
5. Store as micro-dollars: `const microDollars = Math.round(cost * 1_000_000)`
6. Increment storage: `await this.storage.increment('budget', 0, { prompt: microDollars, completion: 0 }, monthlyWindow, 1)`
7. Increment local `paidCallCount`.
8. Call `await this.checkThresholds(cost)` to handle events.

**Method: `async getMonthlySpend(): Promise<number>`**

Get current month's total spend in dollars:

1. Call `await this.storage.getUsage('budget', 0, monthlyWindow)`
2. Return `usage.promptTokens / 1_000_000` (convert micro-dollars back to dollars)

**Method: `async isExceeded(): Promise<boolean>`**

1. If `budgetConfig.monthlyLimit === 0`, return false (no budget = no enforcement... wait, $0 means never spend money).

Actually per CONTEXT.md: "$0 budget prevents any paid calls". So:

- If `budgetConfig.monthlyLimit === 0`: return `true` (zero budget = all paid calls blocked)
- Otherwise: `return (await this.getMonthlySpend()) >= this.budgetConfig.monthlyLimit`

**Method: `async getRemainingBudget(): Promise<number>`**

1. If `budgetConfig.monthlyLimit === 0`, return `0`.
2. `const spent = await this.getMonthlySpend()`
3. `return Math.max(0, this.budgetConfig.monthlyLimit - spent)`

**Private method: `async checkThresholds(lastRequestCost: number): Promise<void>`**

1. Get current spend: `const spent = await this.getMonthlySpend()`
2. If `this.budgetConfig.monthlyLimit <= 0`, return (no thresholds to check with $0 budget).
3. Calculate ratio: `const ratio = spent / this.budgetConfig.monthlyLimit`
4. For each threshold in `this.budgetConfig.alertThresholds`:
   - If `ratio >= threshold` and `!this.alertsFired.has(threshold)`:
     - Add threshold to alertsFired
     - Emit `budget:alert` event via `this.emitter.emit(RouterEvent.BUDGET_ALERT, payload)`
     - Payload: `{ threshold, spent, limit: this.budgetConfig.monthlyLimit, remaining: Math.max(0, this.budgetConfig.monthlyLimit - spent), avgCostPerRequest: this.paidCallCount > 0 ? spent / this.paidCallCount : 0, timestamp: Date.now() }`
5. If `spent >= this.budgetConfig.monthlyLimit`:
   - Emit `budget:exceeded` event
   - Payload: `{ spent, limit: this.budgetConfig.monthlyLimit, lastRequestCost, timestamp: Date.now() }`

Wrap all `emitter.emit()` calls in try/catch (fire-and-forget pattern).

**Method: `resetAlerts(): void`**

Clear `alertsFired` Set and reset `paidCallCount` to 0. Called on month boundary or manually.
