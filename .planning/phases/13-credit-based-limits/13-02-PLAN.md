**Storage key pattern:** `credit:{provider}` with keyIndex 0, consumed amount stored as micro-dollars in promptTokens field. Use a 100-year durationMs to prevent storage expiration (credits don't reset):

```typescript
const CREDIT_WINDOW: TimeWindow = {
  type: 'monthly',
  durationMs: 100 * 365 * 24 * 60 * 60 * 1000, // 100 years, never expires
};
```

**Constructor:**

```typescript
constructor(
  storage: StorageBackend,
  cooldownManager: CooldownManager,
  emitter: EventEmitter,
  configs: Map<string, CreditConfig>,
)
```

- Store all deps as private readonly
- Initialize `alertsFired: Map<string, Set<number>>` (per-provider dedup)
- Initialize `consumedCache: Map<string, number>` (in-memory cache of consumed micro-dollars)
- Initialize `balances: Map<string, number>` (mutable copy of each provider's balance for topUp support -- do NOT mutate the CreditConfig objects directly as they may be Zod-inferred readonly types)

**loadPersistedCredits():**

- For each provider in configs, call `storage.getUsage('credit:{provider}', 0, CREDIT_WINDOW)`
- Store result.promptTokens in consumedCache (this is micro-dollars)
- Initialize balances map from configs: `balances.set(provider, config.balance)`
- Check expiry warnings: if expiresAt is within expiryWarningDays, emit credit:expiring event
- Fire-and-forget pattern: wrap in try/catch, log errors with debug

**recordConsumption(provider, usage):**

- Get config for provider. If not found, return (no-op for non-credit providers)
- Calculate cost: `(usage.promptTokens * costRates.inputPer1MTokens + usage.completionTokens * costRates.outputPer1MTokens) / 1_000_000`
- Convert to micro-dollars: `Math.round(cost * 1_000_000)`
- Persist: `storage.increment('credit:{provider}', 0, { prompt: microDollars, completion: 0 }, CREDIT_WINDOW, 1)`
- Update consumedCache: add microDollars to existing value
- Check thresholds and emit credit:low events
- Fire-and-forget: wrap storage call in try/catch, debug log on failure
- After recording, check if estimated remaining <= 0: log debug message but DO NOT mark depleted yet (wait for 402 confirmation per locked decision "try one more, then confirm")

**confirmExhaustion(provider):**

- Called when a 402 is received for a provider with credit config
- Mark provider as permanently depleted: `cooldownManager.setProviderCooldown(provider, Infinity, 'permanent', 'Credits exhausted')`
- Emit credit:exhausted event with reason: 'depleted'
- Debug log: "Provider %s credits confirmed exhausted (402 received)"

**isEstimatedExhausted(provider):**

- Get config and consumed from cache
- Get balance from balances map (not config.balance -- topUp may have changed it)
- Calculate: `remaining = balance * 1_000_000 - consumed`
- Return remaining <= 0

**isExpired(provider):**

- Get config. If no expiresAt, return false
- Return `Date.now() > new Date(config.expiresAt).getTime()`

**shouldSkip(provider):**

- If not in configs map, return false (non-credit provider)
- Check isExpired first. If expired and not already depleted:
  - Mark depleted via CooldownManager
  - Emit credit:exhausted with reason: 'expired'
  - Return true
- If isEstimatedExhausted, return false (per "try one more" strategy -- let the call go through, 402 will confirm)
- Return false

**topUp(provider, amount):**

- Get config. If not found, throw ConfigError
- Increase balance in the balances map (NOT config.balance): `balances.set(provider, (balances.get(provider) ?? config.balance) + amount)`
- If provider is depleted, clear depletion: `cooldownManager.onProviderSuccess(provider)`
- Clear alertsFired for this provider (alerts may need re-firing at new thresholds)
- Debug log: "Provider %s topped up by $%d, new balance: $%d"

**getStatus(provider):**

- Get config and consumed from cache
- Get balance from balances map
- Return CreditStatus object with all computed fields
- Use conditional property inclusion for optional expiresAt field
- Return undefined if provider not in configs

**checkThresholds(provider, consumed):** (private)

- Get balance from balances map
- Calculate percentRemaining = (balance _ 1M - consumed) / (balance _ 1M)
- For each threshold in alertThresholds, if percentRemaining <= threshold and not in alertsFired set:
  - Add threshold to alertsFired set
  - Emit credit:low event via safeEmit pattern (try/catch around emitter.emit)
