**Update `classifyError()` function:**

1. For `statusCode === 429`:
   - Parse retry-after header (already done). Compute `cooldownMs` from it.
   - If retry-after is present and parseable as seconds: `cooldownMs = seconds * 1000`
   - If retry-after is present and parseable as HTTP-date: `cooldownMs = Math.max(0, date.getTime() - Date.now())`
   - If retry-after is missing or unparseable: `cooldownMs = undefined` (caller will use default)
   - Set `cooldownClass`: if `cooldownMs !== undefined && cooldownMs < 120_000` then `'short'`, else if `cooldownMs !== undefined` then `'long'`, else `undefined` (unknown, caller decides)
   - Also check `x-ratelimit-reset` headers as fallback: parse `error.responseHeaders?.['x-ratelimit-reset-requests']` or `error.responseHeaders?.['x-ratelimit-reset']`. These are often epoch seconds or ISO dates.

2. For `statusCode === 402`:
   - Return with `type: 'rate_limit'` (same bucket for downstream handling)
   - `cooldownMs: Infinity` (permanent)
   - `cooldownClass: 'permanent'`
   - `retryable: false` (do NOT retry 402)
   - `message`: include "credits exhausted" context

3. Keep ALL existing behavior unchanged for 401, 403, 5xx, network errors.

**Add helper function:**

```typescript
function parseRetryAfter(header: string | undefined): number | undefined {
  if (!header) return undefined;
  const seconds = Number(header);
  if (!isNaN(seconds) && seconds >= 0) return seconds * 1000;
  const date = new Date(header);
  if (!isNaN(date.getTime())) return Math.max(0, date.getTime() - Date.now());
  return undefined;
}

function parseRateLimitReset(headers: Record<string, string> | undefined): number | undefined {
  if (!headers) return undefined;
  // Try x-ratelimit-reset-requests first (Groq, SambaNova), then x-ratelimit-reset (Google)
  const resetValue = headers['x-ratelimit-reset-requests'] ?? headers['x-ratelimit-reset'];
  if (!resetValue) return undefined;
  // Could be epoch seconds or ISO date
  const asNumber = Number(resetValue);
  if (!isNaN(asNumber)) {
    // If looks like epoch (> year 2000 in seconds), treat as absolute
    if (asNumber > 1_000_000_000) return Math.max(0, asNumber * 1000 - Date.now());
    // Otherwise treat as relative seconds
    return asNumber * 1000;
  }
  const date = new Date(resetValue);
  if (!isNaN(date.getTime())) return Math.max(0, date.getTime() - Date.now());
  return undefined;
}
```

**Extend CooldownManager** with provider-level cooldown tracking alongside existing per-key tracking.

The key distinction: per-key cooldowns are for individual keys hitting 429. Provider-level cooldowns are for when ALL keys for a provider are exhausted (set by ChainExecutor, not by retry proxy).

**Add these new methods:**

```typescript
private providerCooldowns: Map<string, { until: number; reason: string; cooldownClass: CooldownClass }>;
private providerFailures: Map<string, number>;
```

Update MAX_COOLDOWN_MS from 5 minutes to 15 minutes (per CONTEXT.md user decision):

```typescript
const MAX_COOLDOWN_MS = 15 * 60 * 1000; // 15 minutes (user decision)
```

**New methods:**

1. `setProviderCooldown(provider: string, cooldownMs: number, cooldownClass: CooldownClass, reason?: string): void`
   - If cooldownClass is 'permanent', set `until: Infinity` — provider is depleted for session
   - Otherwise: apply escalating backoff. Get `providerFailures` count for this provider. Compute `effectiveCooldown = Math.min(cooldownMs * Math.pow(2, failures), MAX_COOLDOWN_MS)`. Set `providerCooldowns.set(provider, { until: Date.now() + effectiveCooldown, reason, cooldownClass })`. Increment `providerFailures`.

2. `isProviderInCooldown(provider: string): boolean`
   - Check `providerCooldowns` map. Lazy cleanup if expired. Return true/false.

3. `getProviderCooldown(provider: string): { until: string; reason: string; cooldownClass: CooldownClass } | null`
   - Return cooldown details or null if no active cooldown.

4. `onProviderSuccess(provider: string): void`
   - Reset `providerFailures` count for provider to 0. Delete `providerCooldowns` entry.

5. `isProviderDepleted(provider: string): boolean`
   - Returns true if provider has a permanent cooldown (cooldownClass === 'permanent').

6. Update `clearAll()` to also clear `providerCooldowns` and `providerFailures`.

