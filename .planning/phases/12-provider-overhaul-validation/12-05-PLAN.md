**Core execution function `executeChain`:**

```typescript
async function executeChain(
  callFn: (model: LanguageModelV3) => PromiseLike<unknown>,
  chain: ChainEntry[],
  filter: ChainFilter | undefined,
  deps: ChainExecutorDeps,
  requestId: string,
): Promise<{ result: unknown; chainPosition: number; entry: ChainEntry; attempts: ChainAttempt[]; fallbackUsed: boolean }> {
```

Logic:

1. Filter chain entries by `filter` if provided (capabilities, provider, tier).
2. Walk filtered chain from position 0.
3. For each entry:
   a. Check `entry.stale` -- if true, skip.
   b. Check `deps.cooldownManager.isProviderInCooldown(entry.provider)` -- if true, skip.
   c. Check `!entry.free && await deps.budgetTracker.isExceeded()` -- if true, skip (paid model + budget exhausted).
   d. Create a RetryProxy for this entry (lazy -- create on first attempt, cache by provider). The RetryProxy needs: provider, modelName (apiModelId), modelId, keys from config, registry (use getProviderModule to get factory), cooldownManager, disabledKeys, emitter, requestId, keySelector. **Important**: The RetryProxy creation needs a LanguageModelV3 as `initialModel`. Get this by calling `providerModule.createFactory(firstKey)` then `factory(apiModelId)`. This is an async operation.
   e. Try `await callFn(retryProxy)`.
   f. On success: emit `chain:resolved` event, call `cooldownManager.onProviderSuccess(entry.provider)`, return result with chainPosition and `fallbackUsed = chainPosition > 0`.
   g. On error: classify with `classifyError()`. Record ChainAttempt. If all keys for provider exhausted (error is from RetryProxy exhausting keys), call `cooldownManager.setProviderCooldown()` with the classified cooldown info. If 404, mark `entry.stale = true`. Continue to next entry.
4. If entire chain exhausted: throw `AllProvidersExhaustedError` with all attempts.

**Detecting "all keys exhausted" for a provider:** The RetryProxy throws when all keys are tried. The error will be a ProviderError or AllKeysExhaustedError. Check if the error indicates all keys were tried by inspecting the error type (ProviderError with `errorType === 'rate_limit'` typically means all keys exhausted from retry proxy).

**RetryProxy caching:** Create a Map<string, Map<string, LanguageModelV3>> keyed by `provider:modelId` to cache retry proxies created during chain execution. Each doGenerate/doStream call creates fresh proxies (since key selection may change).

Actually, per CONTEXT.md: "retry proxy rotates keys first: on 429, try all keys for that provider. Only when ALL keys exhausted does chain advance." So we need to create a RetryProxy per chain entry per request.

**Provider factory caching:** Cache the `(modelId: string) => LanguageModelV3` factory per provider+key to avoid repeated dynamic imports. Use a Map<string, (modelId: string) => LanguageModelV3> keyed by `${provider}:${keyHash}`.

**`getChainStatus` function:**

```typescript
export function getChainStatus(chain: ChainEntry[], cooldownManager: CooldownManager): ChainStatus {
  const entries: ChainEntryStatus[] = chain.map((entry) => {
    let status: ChainEntryStatus['status'] = 'available';
    let cooldownUntil: string | undefined;
    let cooldownClass: CooldownClass | undefined;

    if (entry.stale) {
      status = 'stale';
    } else if (cooldownManager.isProviderDepleted(entry.provider)) {
      status = 'depleted';
      cooldownClass = 'permanent';
    } else if (cooldownManager.isProviderInCooldown(entry.provider)) {
      status = 'cooling';
      const cd = cooldownManager.getProviderCooldown(entry.provider);
      if (cd) {
        cooldownUntil = cd.until;
        cooldownClass = cd.cooldownClass;
      }
    }

    const entryStatus: ChainEntryStatus = {
      provider: entry.provider,
      modelId: entry.modelId,
      qualityTier: entry.qualityTier,
      free: entry.free,
      status,
    };
    if (cooldownUntil !== undefined) entryStatus.cooldownUntil = cooldownUntil;
    if (cooldownClass !== undefined) entryStatus.cooldownClass = cooldownClass;
    return entryStatus;
  });

  return {
    entries,
    totalModels: chain.length,
    availableModels: entries.filter((e) => e.status === 'available').length,
    depletedProviders: [
      ...new Set(entries.filter((e) => e.status === 'depleted').map((e) => e.provider)),
    ],
  };
}
```

Actually, the current `ProviderFactory` type is synchronous: `(options: { apiKey: string }) => (modelId: string) => LanguageModelV3`. But provider modules have `async createFactory()`. Two options:

1. Make ProviderFactory async
2. Pre-initialize factories at createRouter time

Go with option 2 to minimize API surface changes. In createRouter, for each configured provider:

1. Get the ProviderModule
2. Call `await mod.createFactory(firstApiKey)` to get the model factory
3. Register `(opts) => { const p = syncFactory(opts.apiKey); return p; }` -- but wait, createFactory is per-key, not per-provider.

```typescript
/**
 * Record a 429/402 rate limit event for observability.
 * Called by ChainExecutor when a provider/key returns 429 or 402.
 * Fire-and-forget pattern — never throws.
 */
recordRateLimitEvent(
  provider: string,
  keyIndex: number,
  cooldownMs: number,
  cooldownTriggered: boolean,
): void {
  try {
    const now = new Date().toISOString();

    // Update per-key stats
    const keyKey = `${provider}:${keyIndex}`;
    const keyStats = this.rateLimitStats.get(keyKey) ?? {
      rateLimitHits: 0, lastRateLimited: null, cooldownsTriggered: 0, totalCooldownMs: 0,
    };
    keyStats.rateLimitHits++;
    keyStats.lastRateLimited = now;
    if (cooldownTriggered) keyStats.cooldownsTriggered++;
    if (cooldownMs !== Infinity) keyStats.totalCooldownMs += cooldownMs;
    this.rateLimitStats.set(keyKey, keyStats);

    // Update per-provider stats
    const provStats = this.providerRateLimitStats.get(provider) ?? {
      rateLimitHits: 0, lastRateLimited: null, cooldownsTriggered: 0, totalCooldownMs: 0,
    };
    provStats.rateLimitHits++;
    provStats.lastRateLimited = now;
    if (cooldownTriggered) provStats.cooldownsTriggered++;
    if (cooldownMs !== Infinity) provStats.totalCooldownMs += cooldownMs;
    this.providerRateLimitStats.set(provider, provStats);

    log('Rate limit event recorded for %s:%d (cooldown: %dms, triggered: %s)',
      provider, keyIndex, cooldownMs, cooldownTriggered);
  } catch (error) {
    log('Failed to record rate limit event: %s', error instanceof Error ? error.message : String(error));
  }
}
```
