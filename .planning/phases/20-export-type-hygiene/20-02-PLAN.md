The emission should happen only when there IS a next entry to fall back to (i.e., `position + 1 < filteredChain.length`).

Add this code block at the end of the catch block, after the `entry.stale = true` block (after line 368) and before the closing `}` of the catch:

```typescript
// Emit fallback:triggered when advancing to next chain entry
const nextPosition = position + 1;
if (nextPosition < filteredChain.length) {
  const nextEntry = filteredChain[nextPosition]!;
  safeEmit(deps.emitter, RouterEvent.FALLBACK_TRIGGERED, {
    fromProvider: entry.provider,
    toProvider: nextEntry.provider,
    fromModel: entry.modelId,
    toModel: nextEntry.modelId,
    reason: classified.message,
    timestamp: Date.now(),
    requestId,
  });
}
```

This uses the existing `safeEmit` helper (line 93) and the already-imported `RouterEvent` constant (line 22). No new imports needed.

The payload matches the `FallbackTriggeredEvent` interface:

- `fromProvider`: current entry's provider that failed
- `toProvider`: next entry's provider that will be tried
- `fromModel`: current entry's modelId
- `toModel`: next entry's modelId
- `reason`: the classified error message
- `timestamp`: standard event timestamp
- `requestId`: for correlation

Note: The event fires even if the next entry will be skipped (stale, cooldown, etc.). This is correct -- it signals the chain is ATTEMPTING to fall back. The chain:resolved event already reports the final outcome.
